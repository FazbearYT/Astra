# Astra ML — подсистема адаптации моделей под рабочие профили

Веб-приложение для автоматического выбора модели машинного обучения под
конкретный табличный датасет. Программа строит **профиль данных**
(распределения, выбросы, баланс классов, корреляции), сопоставляет
профиль с требованиями каждой модели из портфеля, **адаптирует
гиперпараметры** под выявленные характеристики и выбирает лучшую модель
по гибридной метрике (75% качество × 25% соответствие профилю).

> Учебный проект в рамках учебно-технологической проектной практики
> (Eltex / РиМ / Astra Linux). Тема: **«Подсистема адаптации моделей под
> рабочие профили»**.

## Что умеет

- Загрузка пользовательского CSV (до 50 МБ / 500 000 строк) с
  авто-определением кодировки, разделителя и санитизацией колонок.
- Встроенный генератор тестовых датасетов (Iris, Wine, Digits, Titanic,
  Synthetic, Large) — Iris/Titanic/Wine создаются автоматически при
  первом запуске.
- Предобработка: импьютация пропусков (median/mode), one-hot и
  label-кодирование категориальных колонок, кодирование строкового
  target с сохранением подписей классов для confusion matrix.
- Pre-flight проверки: ≥30 строк, ≥2 признака, ≥2 класса.
- Профилирование данных через `DataProfiler` (12+ характеристик).
- Портфель из 8 моделей:
  RandomForest, SVM, GradientBoosting, MLP, LogisticRegression,
  KNN, GaussianNB, DecisionTree.
- Profile-aware адаптация гиперпараметров (n_estimators, max_depth,
  hidden_layer_sizes, n_neighbors, kernel и т.д. меняются под размер
  и структуру данных).
- Гибридный финальный скоринг = α × metric_score + (1−α) × profile_score,
  где profile_score реально влияет на выбор лучшей модели.
- Три пресета (Fast / Standard / Accurate) + Advanced-режим с ручной
  настройкой гиперпараметров каждой модели и весов метрик.
- Метрики: Accuracy + F1 (классификация), R²/RMSE (регрессия), опционально
  CV-score.
- Confusion matrix с человеческими подписями классов, сравнительная
  таблица моделей, объяснение «почему выбрана эта модель».
- Скачивание лучшей модели (`.pkl`) и JSON-отчёта. Все артефакты
  сохраняются в `outputs/run_NNN/`.

## Запуск

```bash
python -m pip install -r requirements.txt
streamlit run web_app.py
```

Откроется на `http://localhost:8501`. При первом запуске будут
автоматически созданы три демо-датасета в `data/tabular/`.

## Тематический контекст

Задача относится к классу **algorithm selection problem** (Rice, 1976):
по характеристикам входной задачи выбрать наиболее подходящий алгоритм
из конечного портфеля. В отличие от AutoML-систем
(auto-sklearn / PyCaret / FLAML), которые делают перебор гиперпараметров
байесовским оптимизатором, эта подсистема использует **правило-ориентированный
подход**: явные требования каждой модели к профилю, прозрачное правило
ранжирования, интерпретируемое решение «почему выбрана именно эта модель».

Сильные стороны:

- Прозрачность: для каждой выбранной модели можно показать, какие
  характеристики профиля её «утвердили».
- Скорость: нет итеративной HPO, обучение происходит за один проход.
- Малая ресурсоёмкость: ноль внешних зависимостей кроме `scikit-learn`,
  `pandas`, `streamlit`, `plotly`.

Ограничения:

- Правила `matches_profile` и `adapt_params_to_profile` — экспертные,
  не выученные. Развитие в сторону обученного meta-learner — отдельная
  тема магистерского уровня.
- Текущая поддержка ограничена табличными датасетами и стандартными
  scikit-learn моделями.
